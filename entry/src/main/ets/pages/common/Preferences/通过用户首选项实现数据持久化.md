# 通过用户首选项实现数据持久化

---

## 一、概述与适用场景

### 1.1 什么是用户首选项

用户首选项为应用提供 **Key-Value 键值型** 的数据处理能力，支持应用持久化轻量级数据，并对其修改和查询。

| 特性         | 说明                                    |
| ------------ | --------------------------------------- |
| **存储形式** | 键值对                                  |
| **存储位置** | 应用沙箱内部（`preferencesDir` 路径下） |
| **数据缓存** | 全量加载到内存，读取速度快              |
| **持久化**   | 调用 `flush()` 将内存数据写入文件       |

### 1.2 适用场景

| 场景类型           | 具体示例                                   |
| ------------------ | ------------------------------------------ |
| **用户个性化设置** | 字体大小、主题颜色、夜间模式开关           |
| **应用配置信息**   | 语言设置、通知偏好、音量设置               |
| **用户状态记录**   | 是否同意隐私协议、登录状态、引导页显示标记 |
| **轻量级缓存**     | 最近搜索记录、临时筛选条件                 |

### 1.3 不适用场景

- 大量数据存储（建议数据不超过 **1万条**）
- 复杂关系型数据（应使用关系型数据库）
- 多进程并发访问场景（XML模式不支持）
- 需要加密存储的敏感数据

## 二、运作机制

```
┌─────────────────────────────────────────────────────────────┐
│                      用户应用程序                             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    ArkTS 接口调用                            │
│           getPreferences / put / get / flush                │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   Preferences 实例                           │
│              （内存缓存 + 文件持久化）                         │
└─────────────────────────────────────────────────────────────┘
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
            ┌─────────────┐     ┌─────────────┐
            │   内存缓存   │     │  持久化文件  │
            │  （高速读写） │     │   （XML格式） │
            └─────────────┘     └─────────────┘
```

**核心流程**：
1. 应用启动时，将持久化文件数据**全量加载**到内存
2. 所有读写操作直接在**内存**中进行（高速访问）
3. 调用 `flush()` 时，将内存数据**同步写入**持久化文件

## 三、约束限制

### 3.1 数据限制

| 限制项         | 限制值                            |
| -------------- | --------------------------------- |
| Key 最大长度   | **1024 字节**                     |
| Value 最大长度 | **16MB**（string类型，UTF-8编码） |
| 建议数据量     | **不超过 1万条**                  |
| 建议总数据量   | **不超过 50MB**                   |

### 3.2 使用限制

| 限制项           | 说明                                                    |
| ---------------- | ------------------------------------------------------- |
| **不支持多进程** | XML模式无法保证进程并发安全，会有文件损坏和数据丢失风险 |
| **不支持加密**   | 如需加密，应用应先加密数据再存储                        |
| **内存开销**     | 数据量越大，内存占用越大                                |

### 3.3 存储模式（API 18+）

| 模式                | 说明                                       | 适用场景         |
| ------------------- | ------------------------------------------ | ---------------- |
| **XML模式**（默认） | 数据以XML形式存储，需调用flush持久化       | 单进程、小数据量 |
| **GSKV模式**        | 数据以二进制存储，实时落盘，支持多进程并发 | 多进程并发场景   |

## 四、核心接口说明

### 4.1 导入模块

```typescript
import { preferences } from '@kit.ArkData';
```

### 4.2 主要接口

| 接口名称                               | 描述                             |
| -------------------------------------- | -------------------------------- |
| `getPreferencesSync(context, options)` | 获取 Preferences 实例            |
| `putSync(key, value)`                  | 将数据写入 Preferences 实例      |
| `getSync(key, defValue)`               | 获取键对应的值，不存在返回默认值 |
| `hasSync(key)`                         | 检查是否包含指定 Key             |
| `deleteSync(key)`                      | 删除指定键值对                   |
| `flush(callback)`                      | 将数据持久化到文件               |
| `on('change', callback)`               | 订阅数据变更                     |
| `deletePreferences(context, options)`  | 删除 Preferences 实例及文件      |

### 4.3 支持的数据类型

```typescript
// 基本类型
preferences.putSync('string_key', 'Hello');
preferences.putSync('number_key', 100);
preferences.putSync('boolean_key', true);

// 数组类型
preferences.putSync('array_number', [1, 2, 3]);
preferences.putSync('array_string', ['a', 'b', 'c']);
preferences.putSync('array_boolean', [true, false]);

// Uint8Array（用于特殊字符）
let uInt8Array = new util.TextEncoder().encodeInto('~！@#￥%');
preferences.putSync('binary_key', uInt8Array);
```

## 五、完整开发步骤

### 5.1 步骤一：获取 Preferences 实例

**在 EntryAbility 中初始化**：

```typescript
// EntryAbility.ets
import { UIAbility, AbilityConstant, Want } from '@kit.AbilityKit';
import { preferences } from '@kit.ArkData';
import { window } from '@kit.ArkUI';

let dataPreferences: preferences.Preferences | null = null;

export default class EntryAbility extends UIAbility {
  onWindowStageCreate(windowStage: window.WindowStage): void {
    // 获取 Preferences 实例
    let options: preferences.Options = { name: 'myStore' };
    dataPreferences = preferences.getPreferencesSync(this.context, options);
    
    // 存储到全局，供其他页面使用
    AppStorage.setOrCreate('preferences', dataPreferences);
  }
}
```

### 5.2 步骤二：写入数据

```typescript
// 写入数据前检查 Key 是否存在
if (dataPreferences.hasSync('startup')) {
  console.info("Key 'startup' 已存在");
} else {
  // 写入数据到内存缓存
  dataPreferences.putSync('startup', 'auto');
  dataPreferences.putSync('fontSize', 16);
  dataPreferences.putSync('darkMode', false);
  
  // 持久化到文件（重要！）
  dataPreferences.flush((err) => {
    if (err) {
      console.error(`持久化失败: ${err.message}`);
      return;
    }
    console.info('数据持久化成功');
  });
}
```

### 5.3 步骤三：读取数据

```typescript
// 读取数据，第二个参数为默认值
let startup: string = dataPreferences.getSync('startup', 'default');
let fontSize: number = dataPreferences.getSync('fontSize', 14);
let darkMode: boolean = dataPreferences.getSync('darkMode', false);

console.info(`startup: ${startup}, fontSize: ${fontSize}, darkMode: ${darkMode}`);
```

### 5.4 步骤四：删除数据

```typescript
// 删除指定 Key
dataPreferences.deleteSync('startup');

// 持久化删除操作
dataPreferences.flush();
```

### 5.5 步骤五：订阅数据变更

```typescript
// 定义观察者回调
let observer = (key: string) => {
  console.info(`Key '${key}' 发生了变化`);
};

// 订阅数据变更
dataPreferences.on('change', observer);

// 修改数据并持久化（会触发回调）
dataPreferences.putSync('startup', 'manual');
dataPreferences.flush(); // flush 后触发 observer 回调

// 取消订阅
dataPreferences.off('change', observer);
```

### 5.6 步骤六：删除 Preferences 文件

```typescript
// 删除 Preferences 实例及持久化文件
let options: preferences.Options = { name: 'myStore' };
preferences.deletePreferences(context, options, (err) => {
  if (err) {
    console.error(`删除失败: ${err.message}`);
    return;
  }
  console.info('删除成功');
});
```

## 六、工具类封装（最佳实践）

### 6.1 封装 PreferencesUtil 工具类

```typescript
// utils/PreferencesUtil.ets
import { preferences } from '@kit.ArkData';
import { BusinessError } from '@kit.BasicServicesKit';

export class PreferencesUtil {
  private static instance: PreferencesUtil;
  private dataPreferences: preferences.Preferences | null = null;
  
  private readonly STORE_NAME: string = 'app_preferences';

  // 单例模式
  public static getInstance(): PreferencesUtil {
    if (!PreferencesUtil.instance) {
      PreferencesUtil.instance = new PreferencesUtil();
    }
    return PreferencesUtil.instance;
  }

  // 初始化（在 EntryAbility 中调用）
  public init(context: Context): void {
    let options: preferences.Options = { name: this.STORE_NAME };
    this.dataPreferences = preferences.getPreferencesSync(context, options);
    
    // 订阅数据变更
    this.dataPreferences.on('change', (key: string) => {
      console.info(`[Preferences] Key '${key}' 已变更`);
    });
  }

  // 保存数据
  public put(key: string, value: preferences.ValueType): void {
    if (!this.dataPreferences) return;
    this.dataPreferences.putSync(key, value);
    this.dataPreferences.flush();
  }

  // 获取数据
  public get(key: string, defaultValue: preferences.ValueType): preferences.ValueType {
    if (!this.dataPreferences) return defaultValue;
    return this.dataPreferences.getSync(key, defaultValue);
  }

  // 删除数据
  public delete(key: string): void {
    if (!this.dataPreferences) return;
    this.dataPreferences.deleteSync(key);
    this.dataPreferences.flush();
  }

  // 检查 Key 是否存在
  public has(key: string): boolean {
    if (!this.dataPreferences) return false;
    return this.dataPreferences.hasSync(key);
  }

  // 清空所有数据
  public clear(): void {
    if (!this.dataPreferences) return;
    this.dataPreferences.clearSync();
    this.dataPreferences.flush();
  }

  // 获取所有数据
  public getAll(): Record<string, preferences.ValueType> {
    if (!this.dataPreferences) return {};
    return this.dataPreferences.getAllSync() as Record<string, preferences.ValueType>;
  }
}

// 导出单例实例
export const preferencesUtil = PreferencesUtil.getInstance();
```

### 6.2 在 EntryAbility 中初始化

```typescript
// EntryAbility.ets
import { preferencesUtil } from '../utils/PreferencesUtil';

export default class EntryAbility extends UIAbility {
  onWindowStageCreate(windowStage: window.WindowStage): void {
    // 初始化 Preferences 工具类
    preferencesUtil.init(this.context);
    
    windowStage.loadContent('pages/Index');
  }
}
```

### 6.3 在页面中使用

```typescript
// pages/Index.ets
import { preferencesUtil } from '../utils/PreferencesUtil';

@Entry
@Component
struct Index {
  @State fontSize: number = 14;
  @State darkMode: boolean = false;

  aboutToAppear(): void {
    // 读取保存的设置
    this.fontSize = preferencesUtil.get('fontSize', 14) as number;
    this.darkMode = preferencesUtil.get('darkMode', false) as boolean;
  }

  build() {
    Column() {
      Text(`字体大小: ${this.fontSize}`)
        .fontSize(this.fontSize)
      
      Button('增大字体')
        .onClick(() => {
          this.fontSize += 2;
          preferencesUtil.put('fontSize', this.fontSize);
        })

      Toggle({ type: ToggleType.Switch, isOn: this.darkMode })
        .onChange((isOn: boolean) => {
          this.darkMode = isOn;
          preferencesUtil.put('darkMode', isOn);
        })
    }
  }
}
```

## 八、常见问题与解决方案

### 8.1 数据丢失问题

**问题**：应用重启后数据丢失

**原因**：写入数据后未调用 `flush()` 持久化

**解决**：
```typescript
// 错误写法
dataPreferences.putSync('key', 'value'); // 仅写入内存

// 正确写法
dataPreferences.putSync('key', 'value');
dataPreferences.flush(); // 必须调用 flush 持久化
```

### 8.2 多进程数据不一致

**问题**：多进程访问导致数据损坏

**解决**：使用 GSKV 存储模式（API 18+）
```typescript
let options: preferences.Options = { 
  name: 'myStore',
  storageType: preferences.StorageType.GSKV  // 使用 GSKV 模式
};
```

### 8.3 特殊字符存储

**问题**：非 UTF-8 字符导致文件损坏

**解决**：使用 Uint8Array 存储
```typescript
import { util } from '@kit.ArkTS';

// 存储特殊字符
let encoder = new util.TextEncoder();
let uInt8Array = encoder.encodeInto('~！@#￥%');
dataPreferences.putSync('special', uInt8Array);

// 读取特殊字符
let decoder = util.TextDecoder.create('utf-8');
let data = dataPreferences.getSync('special', new Uint8Array(0));
let result = decoder.decodeToString(data as Uint8Array);
```

## 九、总结

| 要点             | 说明                             |
| ---------------- | -------------------------------- |
| ✅ **适用场景**   | 轻量级数据、用户设置、应用配置   |
| ✅ **核心流程**   | getPreferences → put/get → flush |
| ✅ **必须持久化** | 写入后务必调用 `flush()`         |
| ✅ **封装工具类** | 提高代码复用性和可维护性         |
| ❌ **避免滥用**   | 不适合大数据量、复杂关系数据     |
| ❌ **注意限制**   | Key长度≤1024字节，数据量≤1万条   |
